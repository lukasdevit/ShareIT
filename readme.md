# ShareIT

**Self-hosted file upload & sharing platform** вҖ” your own mini Dropbox with ShareX integration, cloud storage, and a full admin panel.

> рҹ”— **Live demo**: [goletz.dev](https://goletz.dev)

---

## What problem does it solve?

Existing file-sharing services are bloated, slow, track you, have arbitrary limits, or require paid subscriptions for basic features like custom domains and short URLs. **ShareIT gives you full control** вҖ” one `docker compose up` and you have a private, blazing-fast file host on your own domain with a clean UI, ShareX screenshot workflow, and optional Backblaze B2 cloud storage.

## Stack

| Layer | Technology |
|---|---|
| **Backend** | Node.js + Fastify (TypeScript) |
| **Frontend** | Next.js 15 + React 19 + Tailwind CSS |
| **Database** | SQLite (zero-config, single-file) |
| **Auth** | JWT (jsonwebtoken) + bcrypt |
| **Storage** | Local filesystem or Backblaze B2 (S3-compatible) |
| **Security** | Helmet, rate limiting, ClamAV virus scanning, login lockout |
| **Reverse proxy** | Caddy (auto Let's Encrypt HTTPS) |
| **Container** | Docker + docker compose (3 services) |
| **Testing** | Vitest + supertest |
| **CI/CD** | GitHub Actions вҶ’ auto-deploy on push to main |

**Flow:** Caddy terminates TLS вҶ’ routes API paths (`/file/*`, `/upload`, `/auth/*`, `/admin/*`) to the Fastify backend, everything else to the Next.js frontend. ## Local setup

```bash
# 1. Clone
git clone https://github.com/lukasdevit/ShareIT.git
cd ShareIT

# 2. Configure
cp .env.example .env
# Generate JWT secret:  openssl rand -hex 32
# Edit .env:  JWT_SECRET=<paste>, ADMIN_USERNAME=admin, ADMIN_PASSWORD=<your-password>

# 3. Dev mode (hot reload for both API + frontend)
docker compose -f docker-compose.dev.yml up -d

# 4. Open
# Frontend:  http://localhost:3001
# API:       http://localhost:3000
```

Default admin login: whatever you set in `.env`. First run auto-seeds the admin user and creates the database.

**Without Docker (dev only):**
```bash
npm install
cd frontend && npm install && cd ..
npm run dev:all   # starts API on :3000 + Next.js on :3001
```

## Deployment approach

```bash
# On your VPS:
git clone https://github.com/lukasdevit/ShareIT.git
cd ShareIT
cp .env.example .env   # fill in secrets
DOMAIN= "your.domain.com" docker compose up -d --build
```

Caddy automatically obtains and renews Let's Encrypt SSL certificates. Make sure ports 80 and 443 are open.

**CI/CD pipeline** (`.github/workflows/node.js.yml`):
1. Push to `main` вҶ’ GitHub Actions runs `npm test` (Vitest)
2. Tests pass вҶ’ SSH into VPS вҶ’ `git pull` вҶ’ `docker compose up -d --build`
3. Prune old Docker images

**DB backups** вҖ” add `scripts/backup-db.sh` to cron:
```bash
0 3 * * * /path/to/ShareIT/scripts/backup-db.sh /path/to/backups
```

## Challenges solved

| Challenge | Solution |
|---|---|
| **Streaming uploads to B2 without buffering** | Used `@aws-sdk/lib-storage` + temp files вҖ” stream to disk first for virus scan + size validation, then upload to B2 |
| **Same-domain API routing in prod** | Caddy proxies `/file/*`, `/upload`, `/auth/*`, `/admin/*` to API container; Next.js handles the rest. No CORS, no separate API subdomain |
| **ShareX config generation** | `GET /sharex/config` returns a dynamically generated `.sxcu` file with the user's JWT embedded вҖ” zero manual config for end users |
| **Secure file serving for private files** | `GET /file/:filename` checks JWT ownership if file is private, streams directly otherwise. No redirects, no temporary URLs |
| **Login brute-force protection** | Custom lockout вҖ” track failed attempts per username in SQLite, lock account for 15 minutes after 5 failures |
| **Expiring file cleanup** | `setInterval` every hour runs a `DELETE` query for files past `expires_at`, removes from storage too. No cron dependency |
| **SQLite concurrency** | Single-file DB with WAL mode вҖ” fast reads, safe writes, zero administration. Perfect for single-server deployments |

## API

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/auth/register` | No | Create account |
| `POST` | `/auth/login` | No | Get JWT |
| `GET` | `/auth/me` | Token | Current user |
| `GET` | `/auth/storage` | Token | Storage used/limit |
| `POST` | `/auth/change-password` | Token | Change password |
| `POST` | `/upload` | Token | Upload file (multipart) |
| `POST` | `/sharex/upload` | Token | ShareX-compatible upload |
| `GET` | `/sharex/config` | Token | Auto-generated ShareX config |
| `GET` | `/files?page=1&limit=50&search=` | Token | Paginated file list |
| `PATCH` | `/file/:id` | Token | Toggle public/private |
| `GET` | `/file/:filename` | No* | Serve file (public or owner) |
| `DELETE` | `/file/:id` | Token | Delete file |
| `GET` | `/admin/users` | Admin | User list with storage stats |
| `PATCH` | `/admin/users/:id` | Admin | Edit user (quota, admin flag) |
| `DELETE` | `/admin/users/:id` | Admin | Delete user + files |
| `POST` | `/admin/db` | Admin | Execute SQL query |
| `GET` | `/health` | No | Uptime check |
